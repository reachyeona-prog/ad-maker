# 프랜차이즈 광고 소재 메이커 — 유지보수 가이드

페이히어 프랜차이즈 DB 솔루션 캐로셀 광고(Meta) 소재를 브랜드별로 빠르게 찍어내기 위한 단일 HTML 파일(`index.html`) 정적 도구.

배너 메이커(`/Users/yeona/Desktop/01. AI/banner-maker/`)와 디자인 시스템·렌더 패턴은 공유하지만 **입력·출력 단위가 다른 별도 프로젝트**. 절대 banner-maker 폴더에 변경을 commit하지 않음.

> 라인 번호는 편집할 때마다 살짝 밀릴 수 있어요. 검색 앵커(섹션 헤더 코멘트, 함수명)를 우선 사용하세요.

## 디자인 정체성

배너 메이커가 "Design Spec Sheet" 톤이라면, 광고 메이커는 **"Hooking Tabloid"** 톤 — Meta 피드에서 스크롤을 멈추게 만드는 자극적·도발적 결.

- **Palette**:
  - **Hook colors**: `--hook-yellow #F5FF00`(형광 노랑), `--hook-green #00FF6E`(네온 그린), `--hook-red #E11D2A`(도장 빨강)
  - **Surface**: 사진 풀블리드 + `--scrim rgba(0,0,0,.78)` 어두운 반투명 박스 / `--paper #FFFFFF` 흰 띠
  - **Editor chrome**: 배너 메이커 톤 그대로 — `--bg #F4F1EA` paper, `--ink #1A1A1A` (편집 UI는 정적 결)
- **Type**:
  - 캐로셀 본문: Pretendard ExtraBold/Black + Noto Sans KR (한글 후킹 카피의 굵은 결이 핵심)
  - 편집 UI 메타값: JetBrains Mono (배너 메이커와 동일)
- **Detail**:
  - 표지: 큰 stencil-feel 타이틀 + 형광 키워드 + 빨간 도장 라벨
  - 슬라이드: 사진 위 그라디언트 스크림 + 어두운 박스 / 흰 띠 + ✅ 불릿 (강조 키워드는 형광 컬러)

| 패턴 | 표지 | 슬라이드 | 비고 |
| --- | --- | --- | --- |
| **A** (기본) | 어두운 음식 사진 + 큰 형광 타이틀 + 빨간 도장 라벨 | 사진 풀블리드 + 좌하단 어두운 반투명 박스 (브랜드명 큼직 + 강조 키워드 컬러) | v1 우선 |

| **D** | 말풍선 컴포지션 + 카피 | 사진 + 상단 흰 띠(브랜드명 + 로고) + 하단 검정 박스(✅ bullets) | v2 |

## 입력 모델

배너 메이커는 SPECS의 슬롯 = 정해진 박스 좌표였다면, 여기는 **캠페인 단위 가변 입력**.

```
캠페인
├─ 비율: 1:1 (1080×1080) | 9:16 (1080×1920)
├─ 패턴: A | B | C | D | E
├─ 표지 (1장)
│   ├─ 메인 카피 (후킹 헤드라인)
│   ├─ 서브 카피
│   ├─ 강조 라벨 (선택, 도장 스타일)
│   └─ 배경 이미지
└─ 슬라이드 (가변 N장, +/− 버튼)
    ├─ 브랜드명
    ├─ 브랜드 로고 (업로드, 선택)
    ├─ 배경 이미지 (음식/매장)
    └─ 강점 bullets (2~3개, 강조 키워드는 컬러 토큰으로 마크업)
```

상태는 단일 객체 `state.campaign`로 관리. 슬라이드 배열은 `state.campaign.slides[]`. localStorage에 캠페인 자동 저장(다음 세션에서 복원 가능).

## 자주 하는 작업

### 1) 새 캐로셀 패턴 추가

1. `LAYOUTS[패턴키]`에 `cover(state)` / `slide(brand, state)` 두 함수 추가 (각각 HTMLElement 반환).
2. 패턴 셀렉터 UI에 옵션 등록.
3. 다운로드 함수는 공통 — 패턴별로 분기 안 함.

### 2) 강조 키워드 컬러 마크업

사용자가 카피에 `**굵게**`처럼 마크업하면 토큰화해서 형광 옐로우로 렌더. 구현: `parseEmphasis(text)` → `<span class="hl-yellow">…</span>`. 다른 컬러 토큰: `==그린==`, `~~레드~~` (확장 시).

### 3) 다운로드

- 비율별 정확한 픽셀 출력: 1:1 → 1080×1080, 9:16 → 1080×1920. 화면에는 0.4× 등으로 축소 표시.
- html2canvas: `scale: 1`로 호출 (화면이 이미 정확한 픽셀이면), 또는 작은 미리보기 + `scale: 정확한배수` 조합. 배너 메이커 C1 transform 이슈 참고 — 미리보기 transform scale은 캡처 직전 임시 제거.
- 파일명: `payhere_franchise_{브랜드명}_{slot번호}_{비율}.jpg`
- ZIP 일괄 다운로드는 v2 (JSZip CDN 추가).

### 4) 이미지 업로드 / 미리보기

- 드래그앤드롭 지원 (배너 메이커 `bindDropZone` 패턴 그대로).
- 사진은 슬라이드별 개별. 표지도 별도.
- `object-fit: cover` 대용 처리: 업로드 시 캔버스로 미리 cover-fit 해서 base64로 보관(html2canvas object-fit 미지원 회피).

### 5) 디자인 토큰 변경

`:root` 변수만 수정. 형광 컬러는 `--hook-*`, 어두운 스크림은 `--scrim`.

## 컨벤션 / 주의사항

- **단일 HTML 파일 유지** — 더블클릭 워크플로 우선. 모든 핸들러는 전역 함수, onclick 인라인 OK.
- **banner-maker 코드 직접 import 금지** — 패턴은 참고하되 복사해서 이 프로젝트 안에 둔다.
- **자극적 톤은 의도된 것** — "예쁘게" 다듬는다고 형광 컬러를 무채색으로 바꾸지 않음. 후킹 = KPI.
- **본문 한글 후킹 카피는 ExtraBold/Black 굵기 필수** — Pretendard 800/900 또는 Noto Sans KR Black. 얇으면 후킹 안 됨.
- **html2canvas 한계** — `object-fit` 미지원, CSS `transform: scale` 부모 있으면 레이아웃 깨짐. 캡처 직전 transform 임시 해제.
- **외부 이미지 CORS** — 사용자가 업로드한 base64는 안전. 외부 URL 직접 사용 금지(또는 fetch → base64 변환).
